# Data Directory

This folder contains vocabulary JSON files and conversion utilities for the MachMitDeutsch app.

## Directory Structure

- **cards/** - Source vocabulary files (your original JSON files)
- **\*_converted.json** - App-ready vocabulary files (generated by conversion script)
- **card_template.json** - Template for creating new vocabulary JSON files
- **CARD_TEMPLATE.md** - Comprehensive guide for card creation with LLMs
- **convert-to-app-format.cjs** - Node.js script to convert source files to app format

## Import-Ready Files

These files are ready to import into the app:
- **nouns_v3_150_converted.json** - 193 IT & Business nouns
- **verbs_top100_v1_converted.json** - 100 common verbs
- **phrases_top50_v1_converted.json** - 69 common phrases

Total: **362 vocabulary cards**

## Conversion Workflow

If you have vocabulary files in a different format:

1. Place source files in the `cards/` directory
2. Update `convert-to-app-format.cjs` with your file mappings
3. Run: `node convert-to-app-format.cjs`
4. Import the generated `*_converted.json` files in the app

## App-Compatible JSON Format

```json
{
  "version": "1.0",
  "exported": "2025-11-11T00:00:00Z",
  "cards": [
    {
      "word": "die Schnittstelle",
      "definition": "interface",
      "type": "noun",
      "level": "B1",
      "collection": "IT Vocabulary",
      "tags": ["technical"],
      "gender": "die",
      "plural": "n"
    }
  ]
}
```

**Required fields:** `word`, `definition`, `type`
**Optional fields:** `level`, `collection`, `tags`, type-specific fields

## Type-Specific Fields

### Nouns
- `gender`: "der" | "die" | "das"
- `plural`: plural ending (e.g., "e", "en", "-", "â€“")

### Verbs
- `verbType`: "regular" | "irregular" | "modal" | "separable" | "reflexive"
- `partizipII`: perfect participle (e.g., "gehabt")
- `auxiliary`: "haben" | "sein"
- `separablePrefix`: prefix for separable verbs
- `examples`: { de: "...", en: "..." }

### Phrases
- `context`: when/where to use (e.g., "informal greeting", "business meetings")

## Notes

- Source files in `cards/` are kept for reference and version control
- Converted files use the app's IndexedDB schema
- The app auto-generates `id`, `createdDate`, `cardScore: 0`, and `viewCount: 0` on import
- Duplicate cards (same German word) are automatically skipped during import
